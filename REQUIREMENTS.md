대략 하기와 같은 웹 페이지 기능이 필요한데 이를 위해서 가장 적합한 기술 스택과 대략적인 구성을 선정/가이드 해주고 'AGENTS.md' 로 작성해줘
 - Frontend는 HTML5 표준을 만족해야 함
 - DB는 SQLite3
 - Excel 파일을 업로드&다운로드&편집 할 수 있어야 함
   - 업로드된 Excel 은 DB로 저장되어 관리
   - 다운로드는 DB에 저장된 내용을 Excel로 다운로드 할 수 있음.
   - 실제 Excel과 거의 같은 수준(행 수는 최대 10000개 제한, 시트 수는 최대 64개 제한, 대량 편집 시 배치 저장)의 UX과 동작 편집이 있으면 좋겠음.
   - Excel 서식은 자유로운 필드 정의가 필요
 - 계정은
   - 슈퍼 관리자, 중간 관리자, 일반 사용자 권한이 분리되어야 함
   - 슈퍼 관리자의 초기 계정 및 비밀번호는 admin / admin 으로 해줘
   - 중간 관리자는 슈퍼관리자와 동등한 권한이지만 슈퍼관리자 계정만 수정할 수 없음.
   - 관리자는 계정 정보의 자유로운 추가 필드들 관리
   - 관리자는 계정 정보를 Excel 로 내려받을 수 있고 수정하여 업로드하면 반영될 수 있음.
   - 저장된 DB의 중요 계정에서의 개인 민감한 정보는 암호화되어 저장 관리 되어야 함
   - 관리자는 편집에 제한이 없음.
   - 관리자는 마감 상태를 관리함.
   - 사용자는 관리자가 마감 상태로 전환하기 전까지는 편집이 가능.
 - 보안은 매우 중요한 원칙임
 - 최대한 설치 의존 관계가 적은 것을 선호. (설치 관리 편의성 최선)
 - 모두 공개된 완전 무료인 기술 스택만 사용
 - 대략 동시 접속 사용자 300명 이내 수준.
 - 대략 다음과 같은 시나리오를 만족해야 함
   1. 관리자(중간 관리자 이상)는 계정을 관리
       - 중간 관리자는 최고 관리자를 관리할 수는 없음
   2. 관리자(중간 관리자 이상)는 Excel 서식을 업로드 및 편집을 할 수 있으며 이 때 사용자가 편집 가능한 컬럼과 아닌 컬럼를 정의 할 수 있어야 함 (기본은 모두 수정 가능이지만 관리자가 지정한 필드는 수정 불가하도록)
   3. 관리자(중간 관리자 이상)는 여러가지 서식을 관리(업로드/다운로드/편집/삭제/복사/...)할 수 있으며. 이 중에서 선택해서 이름을 정의하고 workspace로 개시
   4. 등록된 계정의 사용자는 웹을 통해서 동시 접속해서 게시된 workspace excel을 공유된 상태로 수정할 수 있게 됨. 각 사용자가 수정한 내용은 다른 사용자도 동일하게 보여야 함
   5. 최종 관리자가 해당 workspace를 마감을 하면 사용자는 해당 workspace를 수정할 수 없음.
   6. 관리자(중간 관리자 이상)는 계정정보/서식/Workspace등을 언제든지 편집 가능하여, Excel로 다운로드 하여 이를 활용하고 다시 업로드할수도 있으며 업로드하면 반영될 수 있어야 함
